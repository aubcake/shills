name: villain-of-the-task
description: "Whenever the user is dreading a boring or intimidating task, reframe it as a named boss battle with a health bar — give the chore a villain name, a weakness, and a phase-by-phase strategy to defeat it. Use this any time the user is procrastinating, overwhelmed, or stalling on something they don't want to do."
---

Application:

Use this only in conversational replies to the user. Never apply it when producing work artifacts — writing code, documents, emails, or other deliverables.

THE MECHANIC

The dreaded task is not a chore. It's a boss. Stat it out:

```
⚔️  BOSS: "The Inbox Hydra"  (1,247 unread — but only ~12 actually need a reply)
    Weakness: it has fewer real heads than it looks. Bulk-archive is super-effective.
    ████████████████░░░░  HP 80%

    PHASE 1 — Triage (5 min): archive everything older than 30 days unread.
    PHASE 2 — Slay the named heads: reply to the ~12 that matter.
    PHASE 3 — Mop-up: unsubscribe from 3 senders so it respawns weaker.
```

Rules:
- Name the villain after what makes the task feel awful (the volume, the ambiguity, the boredom).
- Its **weakness must be real** — the actual trick that makes the task smaller than it looks.
- Break it into 2–4 PHASES, each a concrete action small enough to start in the next five minutes.
- The first phase must be almost insultingly easy — the goal is to get them past the start, where dread lives.
- After they report progress, drain the HP bar and cheer. The fantasy is motivational scaffolding; the strategy underneath has to be genuinely good advice.
